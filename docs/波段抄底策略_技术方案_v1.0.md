## 三、数据表设计

### 3.1 SQLite 表结构（元数据）

```sql
-- 股票基本信息
CREATE TABLE stock_basic (
    ts_code TEXT PRIMARY KEY,    -- TS代码 (000001.SZ)
    symbol TEXT,                 -- 股票代码
    name TEXT,                   -- 股票名称
    industry TEXT,               -- 所属行业
    list_date DATE,              -- 上市日期
    delist_date DATE,            -- 退市日期（NULL表示未退市）
    status TEXT                  -- 状态 (L上市/D退市/P暂停上市)
);

-- 交易日历
CREATE TABLE trade_calendar (
    exchange TEXT,               -- 交易所 (SSE/SZSE)
    trade_date DATE PRIMARY KEY, -- 交易日期
    is_open INTEGER              -- 是否交易日 (1/0)
);

-- 回测记录
CREATE TABLE backtest_record (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    run_id TEXT UNIQUE,          -- 运行批次ID
    start_date DATE,
    end_date DATE,
    config_json TEXT,            -- 配置快照(JSON)
    created_at TIMESTAMP
);

-- 因子计算缓存索引
CREATE TABLE factor_cache_index (
    factor_name TEXT,
    ts_code TEXT,
    calc_date DATE,
    data_range_start DATE,
    data_range_end DATE,
    file_path TEXT,              -- Parquet文件路径
    PRIMARY KEY (factor_name, ts_code, calc_date)
);
```

### 3.2 Parquet 文件结构（时序行情）

```
/data/
├── daily/                      # 日K线数据
│   ├── stock_000001.SZ.parquet
│   ├── stock_000002.SZ.parquet
│   └── ...
├── adjusted/                   # 后复权数据
│   ├── stock_000001.SZ.parquet
│   └── ...
└── factors/                    # 因子计算结果缓存
    ├── kdj/
    ├── ma/
    ├── rsi/
    └── ...
```

**Parquet Schema（日K线）：**
```
ts_code: string          # 股票代码
trade_date: date         # 交易日期
open: float64            # 开盘价
high: float64            # 最高价
low: float64             # 最低价
close: float64           # 收盘价
volume: int64            # 成交量（手）
amount: float64          # 成交额（千元）
turn: float64            # 换手率（%）
is_suspended: bool       # 是否停牌
is_delisted: bool        # 是否已退市
```

## 五、风险点与应对

### 5.1 数据风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 数据源不稳定 | 无法获取行情数据 | 本地缓存 + 多数据源备用（Tushare/AKShare/东方财富） |
| 复权计算错误 | 回测结果失真 | 对比券商软件复权结果，验证算法 |
| 停牌数据缺失 | 误买入停牌股 | 建立停牌日历，买入前检查 |
| 退市股票处理 | 回测幸存者偏差 | 退市股票纳入回测，退市日清仓 |

### 5.2 技术风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 因子计算精度 | 信号偏差 | 使用 talib 标准实现，交叉验证 |
| 性能瓶颈 | 回测速度慢 | 因子并行计算、Parquet 列式存储 |
| 参数过拟合 | 实盘失效 | 参数敏感性分析、样本外验证 |

### 5.3 业务风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| **权重已修正** | 因子权重完整 | ✅ 已分配：KDJ 45% + 成交量 15% + 均线 15% + RSI 10% + MACD 10% + 布林带 5% = 100% |
| 抄底定义模糊 | 选股信号不一致 | 需求明确抄底条件（如：KDJ<20 且 放量） |
| 回测时间段选择 | 结果偶然性 | 多时间段交叉验证 |
