## 1. 方案概述

## 2. KDJ因子对接方案

### 2.1 方案选择
**采用方案2：使用通达信OHLCV数据计算KDJ**

- **优点**：算法可控、与系统其他因子一致、便于调试
- **可行性**：高
- **预估时间**：1.5小时

### 2.2 KDJ参数确认
与通达信默认参数一致：

| 参数 | 值 | 说明 |
|------|-----|------|
| N | 9 | RSV计算周期 |
| M1 | 3 | K值平滑周期 |
| M2 | 3 | D值平滑周期 |

### 2.3 计算公式验证

#### RSV（未成熟随机值）
```
RSV = (Close - Low_N) / (High_N - Low_N) * 100
```
其中 `High_N` 和 `Low_N` 为N日内最高价和最低价。

#### SMA计算（通达信风格）
```
SMA(X, N) = (Prev_SMA * (N-1) + X) / N
```

#### K、D、J值
```
K = SMA(RSV, M1)
D = SMA(K, M2)
J = 3*K - 2*D
```

### 2.4 算法验证结果

| 测试项 | 结果 | 说明 |
|--------|------|------|
| 参数一致性 | ✅ | N=9, M1=3, M2=3 |
| RSV计算 | ✅ | 公式正确 |
| SMA计算 | ✅ | 通达信风格 |
| J值公式 | ✅ | 3K-2D |
| 真实数据验证 | ✅ | 与通达信数据对比通过 |
| 极端值测试 | ✅ | 超买超卖情况正确 |
| 得分映射 | ✅ | J<20得分100 |

## 3. 数据接口实现

### 3.1 TdxLocalLoader

```python
class TdxLocalLoader:
    """通达信本地数据读取器"""
    
    DAY_RECORD_SIZE = 32  # 每条记录32字节
    
    # struct格式：8个整数 × 4字节
    # I(日期) + IIII(开高低收) + II(成交额、成交量) + I(保留)
    unpack_format = 'IIIIIIII'
```

### 3.2 文件格式

`.day` 文件结构（32字节/条）：

| 字段 | 偏移 | 类型 | 说明 |
|------|------|------|------|
| 日期 | 0-3 | int32 | YYYYMMDD格式 |
| 开盘 | 4-7 | int32 | 实际价格×100 |
| 最高 | 8-11 | int32 | 实际价格×100 |
| 最低 | 12-15 | int32 | 实际价格×100 |
| 收盘 | 16-19 | int32 | 实际价格×100 |
| 成交额 | 20-23 | int32 | 元 |
| 成交量 | 24-27 | int32 | 手 |
| 保留 | 28-31 | int32 | 保留字段 |

1. **数据优先级**：通达信本地数据优先，AKShare兜底

3. **日期格式**：YYYYMMDD整数格式
4. **市场识别**：

   - 深圳：sz（0/3开头）
